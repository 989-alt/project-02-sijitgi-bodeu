## 주요 컴포넌트
1. **AcrosticInput**
   - 머리글자 입력칸: 한 칸짜리 텍스트박스, autoFocus.
   - 입력값 변경 시 `[...글자].filter(c => /\S/.test(c))` 로 줄 카운트 결정.
   - `시작` 누르면 줄 입력칸이 fade-in (200ms, prefers-reduced-motion 존중).
   - 각 줄: `<label>머리글자</label> <input type="text" maxLength="40">` + `글자수 / 음절수` 카운터 (한글 음절은 [가-힣] 개수, 그 외는 모두 글자).
   - 저장 직전 검증: 최소 1줄 이상 비어있지 않아야. 모두 비면 토스트 "시를 적어주세요".

2. **PoemCard (갤러리 카드)**
   - 크림 배경(#f5f0e8), hairline 보더, padding 24px, rounded 12px.
   - 머리글자(첫 글자 강조) → 줄 텍스트.
   - 작가명·작성일자 footer.
   - hover: shadow 살짝, transform 없음 (layout shift 방지 — 우대 룰).
   - 우측 상단: 케밥 메뉴 → `이미지 저장(PNG)`, `삭제`.
   - 카드 클릭 → 풀스크린 미리보기 (발표 모드 단건).

3. **PresentMode (풀스크린 룰렛 + 발표)**
   - `발표 모드` 버튼 클릭 시 갤러리에서 무작위 1편을 풀스크린으로.
   - 좌·우 화살표 / 키보드 ←→: 이전/다음 작품.
   - ESC 또는 좌상단 X: 종료.
   - 갤러리 비었으면 빈 상태 일러스트 + "아직 작품이 없어요" 안내.

4. **ExportMenu (드롭다운)**
   - `학기 시집 PDF`, `JSON 내보내기`, `JSON 가져오기`.
   - PDF 옵션 모달: 학급명 · 시집 제목 · 발문 한 줄 입력 후 PDF 생성.
   - JSON 가져오기는 파일선택 → 갤러리에 병합(중복 id 제외).

5. **FolderSync (조건부)**
   - File System Access API 지원되면 `폴더 동기화 ▢` 토글 노출.
   - 클릭 → `showDirectoryPicker()` → 권한 받으면 갤러리 변경 시 자동으로 `gallery.json` 덮어쓰기.
   - 미지원 브라우저(Firefox/Safari)에서는 회색 처리 + 툴팁 "현재 브라우저에서 지원되지 않아요".

## 접근성
- 모든 인터랙티브 요소: focus outline 2px solid var(--primary), offset 2px.
- 아이콘 단독 버튼: `aria-label` 필수 (`발표 모드`, `삭제`, `이미지 저장` 등).
- 모달(발표 모드·PDF 옵션): `role="dialog" aria-modal="true"`, ESC 닫기, 포커스 트랩(간단히 첫 / 마지막 element 순환).
- 글자수 카운터: `aria-live="polite"`.
- 색상에 의존한 의미 전달 금지 (저장 성공 = 색 + 텍스트 "저장됨").
- `prefers-reduced-motion: reduce` 면 모든 transition `none`.

## 인터랙션 / 단축키
- 머리글자 input 에서 Enter → `시작`.
- 마지막 줄 input 에서 Ctrl/Cmd + Enter → `저장`.
- 갤러리에서 `R` → 발표 모드(룰렛).
- 발표 모드: `←` 이전, `→` 다음, `Esc` 종료, `Space` 다음.

## 빈 상태 / 에러 상태
- 갤러리 비어있음: 일러스트 + "첫 시를 적어보세요" + `머리글자 입력 칸으로 이동` 버튼.
- localStorage 쿼터 초과(매우 드묾): 토스트 + JSON 내보내기 권유.
- File System Access 실패: 토스트 + 동기화 토글 자동 off.
