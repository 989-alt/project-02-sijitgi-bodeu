# Cycle 3 · 학급(workspace) 분리

## 문제
- 모든 시가 `sijitgi-bodeu.poems.v1` 단일 키에 누적 → 두 학급(5-3, 6-1)을 같은 기기에서 가르치는 전담교사·교생 사용 시 작품이 섞임.
- 학급 정보가 PDF 표지 입력칸에만 존재 → 그 외에는 "학급 = 데이터 없음".

## 해결 모델 — 활성 학급(워크스페이스)
- 학급(class) 엔티티 도입: `{ id, name, note, createdAt }`.
- 항상 **하나의 활성 학급**이 존재 (또는 0개 → 환영 모달).
- 작품 저장소를 학급 ID별로 네임스페이스 분리:
  - `sijitgi-bodeu.classes.v1` — 학급 목록
  - `sijitgi-bodeu.activeClassId.v1` — 활성 학급 id
  - `sijitgi-bodeu.poems.v2.<classId>` — 학급별 작품 배열
- 학급 전환 시 갤러리 통째로 재렌더링. 작품은 학급 컨텍스트 안에서만 보임.
- PDF 표지의 학급명 칸은 활성 학급명으로 prefill (수정은 가능, 작품 메타엔 영향 없음).

## 마이그레이션 (v1 → v2)
- 페이지 진입 시: 기존 `poems.v1` 가 있고 학급이 0개이면 →
  1. "우리반" 학급 자동 생성
  2. 그 학급의 `poems.v2.<id>` 로 기존 작품 이관
  3. `poems.v1` 키 삭제
  4. `activeClassId` = 새 학급 id

## UI 변경
- 헤더에 **학급 칩**: 왼쪽 brand 옆에 `5-3 · 12편` 형태. 클릭 → 학급 관리 모달.
- 학급 관리 모달:
  - 학급 목록 (사용중 배지 / 전환 / 이름변경 / 삭제)
  - "+ 새 학급" 입력
  - 삭제는 confirm 한 번 더.
- **환영 모달** (학급 0개일 때): "선생님, 학급 이름을 알려주세요." → 입력 후 시작.
- 학급이 0개일 때: 입력 폼·발표·내보내기 모두 disabled.

## JSON Import/Export 의미
- Export = 현재 활성 학급의 작품만.
- Import = 현재 활성 학급에 병합. (다른 학급 작품을 끌어오고 싶으면 학급을 먼저 전환.)

## 영향 받는 파일
- `app.js` — 저장 모델·헤더 칩·두 모달 (~90% 변경 없음, 저장/복원만 추상화).
- `index.html` — 학급 칩 + 두 모달 마크업.
- `styles.css` — 칩 / 관리 모달 / 환영 모달.
- `tests/e2e.py` — 각 테스트 시작 시 학급을 명시적으로 seed (localStorage), 새 테스트 1건: "학급 분리".

## 비기능 보장
- 마이그레이션은 1회성·idempotent (poems.v1 가 없으면 no-op).
- 환영 모달은 fully accessible (focus trap 간소화, ESC 닫기 불가 — 학급 없이는 진행 불가).
- 학급명은 1~30자, 공백만은 거부.
