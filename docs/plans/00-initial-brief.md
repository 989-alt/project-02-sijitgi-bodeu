# Day 02 · 시 짓기 보드 — 머리글자 시 / 학기 시집

## 토픽 (출처: 100-vibecoding-topics.md #002)
- **카테고리**: 국어
- **콘텐츠 주제·목표**: 머리글자(아크로스틱) 단어를 받아 줄마다 한 문장씩 시를 완성하는 보드. 학기 동안 작품이 쌓이면 자동으로 우리반 시집(PDF)으로 변환. 발표 시 한 명씩 화면에 띄우기.
- **대상**: 초3~6 (1:1 작성 + TV로 공유 발표)
- **AI 사용**: ✕ (의도적 제외 — 학생 본인의 창작이 핵심)

## 핵심 기능 (토픽 명세)
1. 머리글자 자동 분해 → 줄별 입력 칸 생성
2. 글자수·음절 카운터
3. 작품 갤러리 (localStorage 1차, File System Access API 2차)
4. 학기말 자동 시집 생성 (표지 · 차례 · 작품 N개 · 발문) → PDF
5. 무작위 한 작품 띄우기 (발표 룰렛 겸용)
6. 카드 이미지 저장 (html2canvas) · 인쇄

## 배제 기능 (토픽 명세)
- 학생 작품 서버 업로드 / 외부 공유
- AI 자동 작시 (수업 의도와 정면 충돌)
- 검열 / 필터링 모델

## 기술 결정
- **스택**: **단일 `index.html` + vanilla CSS + vanilla JS**. 단순 토픽이고 외부 의존(html2canvas, html2pdf)을 CDN으로만 가져오면 충분. Vite/React는 오버킬.
- **CDN**: html2canvas, html2pdf.js만 unpkg/jsdelivr에서 로드. Tailwind CDN은 사용하지 않음 (헤드리스 환경 블록 + 디자인 토큰 직접 작성).
- **저장**: localStorage(기본) + File System Access API(폴더 지정 시 자동 미러). 두 방식 어느 쪽도 안 되는 브라우저에선 localStorage 단독.
- **PDF**: html2pdf.js (jsPDF + html2canvas 결합)로 단일 PDF 다운로드. 시집 표지/차례/작품/발문을 한 문서에 페이지 단위로 배치.
- **Gemini**: ✕ (사용 안 함 → 프록시·키 무관)

## 비기능 제약
- 학생 개인정보 외부 전송 절대 금지. localStorage 단독 저장.
- 키보드만으로 모든 흐름 완주 가능해야 함 (입력→저장→갤러리→발표).
- 색 대비 4.5:1 이상, focus state 명시, 모든 인터랙티브 요소에 aria-label.
- GitHub Pages 정적 호스팅에서 그대로 동작 (서버 코드 없음).
